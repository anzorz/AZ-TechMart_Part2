JavaScript Enhancements Documentation

Enhancement 1: Checkbox for "Billing Address Different from Delivery Address"

Description: A checkbox is added on the "Enquire" page that, when checked, dynamically displays additional input fields for entering a billing address different from the delivery address.

Interaction: The user triggers the enhancement by checking the "Billing address different from Delivery Address" checkbox. This action displays additional input fields for the billing address, allowing users to enter the necessary details.

Implementation: The enhancement is implemented using JavaScript, which listens for the "change" event on the checkbox. When the checkbox is checked, the billing address section is displayed, and when unchecked, it is hidden.

Link to Implementation: Enquire Page

References: No third-party sources were used to create this enhancement.

Enhancement 2: Concatenation of First and Last Name to Pre-fill "Name on Credit Card"

Description: On the "Enquire" page, the user's first and last names are concatenated and stored in the session storage. This stored full name is then automatically used to pre-fill the "Name on Credit Card" field on the "Payment" page.

Interaction: The user triggers the enhancement by entering their first and last names on the "Enquire" page. This action automatically concatenates the names and stores them. When the user navigates to the "Payment" page, the stored full name is retrieved and pre-filled in the "Name on Credit Card" field.

Implementation: The enhancement is implemented using JavaScript to listen for input events on the first and last name fields. When either field changes, the names are concatenated and stored in session storage. On the "Payment" page, the stored name is retrieved and used to pre-fill the "Name on Credit Card" field.

Link to Implementation: Payment Page
